# Communication Frameworks

**Purpose**: How to explain architectural decisions effectively.

---

## Structure

- **`structured-answers/`** - How to structure explanations
- **`whiteboard-flow/`** - Visual communication patterns
- **`decision-narratives/`** - Storytelling for decisions

---

## How to Use

These frameworks apply to **all roles**. Use them when explaining decisions.

1. Start with `structured-answers/top-down-communication.md`
2. Use `structured-answers/depth-control.md` to manage detail
3. Apply `structured-answers/architecture-explanation-structure.md` for complex systems

---

## Key Principles

- **Role-agnostic**: Communication skills apply to all roles
- **Audience-aware**: Adjust depth and language for audience
- **Decision-focused**: Lead with decisions and trade-offs
